# 프론트엔드 디렉토리 규칙

## 페이지
- 메인 페이지는 `frontend/src/pages/main/`에 두고 파일명은 `mainpage.tsx`로 통일한다.
- 게임별 페이지는 `frontend/src/pages/games/`에 둔다.
- 공용(비게임) 페이지는 `frontend/src/pages/common/`에 둔다.
- 메인/게임/공용을 같은 폴더에 섞지 않는다.

## 컴포넌트
- 재사용 UI 컴포넌트는 `frontend/src/components/`에 둔다.
- 재사용하지 않는 페이지 전용 UI는 해당 페이지 폴더에 둔다.

## 스타일
- 페이지 전용 스타일은 페이지 옆에 둔다. (예: `frontend/src/pages/games/worldcup.css`)
- 공용 스타일은 `frontend/src/styles/`에 둔다. (필요 시 생성해서 사용)
- 모바일 기준을 우선으로 잡고, 좌우 여백/패딩이 깨지지 않게 유지한다.
- 해상도 기준: 최소 320px, 기본 모바일 우선, 최대 폭은 32.5rem(약 520px) 카드형 레이아웃을 유지한다.
- 브레이크포인트는 40rem(약 640px), 64rem(약 1024px) 기준으로 확장한다.

## 데이터 / 유틸
- 정적/목업 데이터는 `frontend/src/data/`에 둔다.
- 순수 유틸 로직은 `frontend/src/utils/`에 둔다.

## 에셋
- 이미지/아이콘은 `frontend/src/assets/`에 둔다.

## 라우팅
- 페이지 이동/추가 시 `frontend/src/App.tsx`의 import 경로를 갱신한다.

## 주석
- 별도 요청 없이는 주석 블록을 풀거나 제거하지 않는다.

## 절대 중요
- clean 하면 죽
